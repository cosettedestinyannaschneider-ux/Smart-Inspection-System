# 项目阶段演示使用报告

本文档用于当前阶段汇报、答辩和演示准备。它描述已经完成的能力、推荐演示路径、当前限制，以及下一阶段法规知识库与 AI 判断升级方向。

## 1. 项目定位

安全生产社会化服务智检系统面向企业安全检查、隐患图片分析、报告生成和管理端治理场景。当前版本已经从早期演示原型推进到具备认证、权限、模型配置、模板、知识库、备份、报告追溯和自动化测试基线的阶段版本。

系统当前核心价值是：

- 用统一账号体系区分普通用户和管理员。
- 普通用户围绕企业、隐患图片、AI 分析、报告下载完成检查主流程。
- 管理员维护用户、企业、部门、模型配置、知识库、报告模板、日志和备份。
- 报告文件和隐患图片通过后端受控接口访问，减少公开直链风险。
- AI 模型配置由后台维护，API Key 不在前端明文展示。
- 报告引用依据已经具备本地知识条款快照结构，为后续法规知识库和规则判定打基础。

## 3. 推荐演示路径

### 演示前准备

1. 确认后端 `.env` 已配置本地数据库、JWT、模型配置密钥。
2. 确认 MySQL 已启动，`ai_project` 数据库可连接。
3. 确认后台至少存在一个已启用、可用的多模态模型配置。
4. 可选导入 `project/database/demo_seed.sql`，用于演示账号和基础企业。
5. 启动后端和前端 H5。

推荐命令：

```powershell
npm --prefix project/backend start
npm --prefix project/uni-app-frontend run dev:h5
```

### 普通用户演示

1. 登录普通用户账号。
2. 进入企业信息页面，展示企业基础档案。
3. 上传一张或多张安全生产现场图片。
4. 在 AI 分析页面输入检查要求并选择模型。
5. 观察等待态、AI 返回结果和图片预览。
6. 保存或调整分析结果。
7. 下载 Word/PDF 报告。
8. 进入历史记录，查看报告、图片和引用依据。

### 管理员演示

1. 登录管理员账号。
2. 查看工作台真实统计。
3. 查看企业档案和报告记录。
4. 打开模型配置，展示新增、启用、检测和 API Key 脱敏。
5. 打开报告模板，展示默认模板保护和模板上传。
6. 打开知识库，展示分类、文档、条款解析状态。
7. 打开日志，展示真实操作记录。
8. 打开备份页，说明已配置 `MYSQLDUMP_BIN` 时可执行真实手动备份。

## 4. 当前限制

| 限制 | 影响 | 当前处理 |
|---|---|---|
| 知识库数据可能为空 | AI 无法形成可靠法规依据判断 | 前端和报告应提示需补充法规条文或人工复核 |
| 条款尚未规则化 | 不能稳定判断“一般隐患/重大隐患”的严格边界 | 后续通过规则库把条文转为可判定规则 |
| AI 视觉识别有随机性 | 同图多次分析可能出现表达差异 | 后续通过场景门禁、事实抽取、规则匹配收敛 |
| 自定义模板兼容性有限 | 任意 Word 模板可能缺少占位符导致报告不完整 | 后续增加模板兼容性检测 |
| Docker PDF 效果弱于 Windows Word | Linux 容器内不能使用 Microsoft Word 自动化 | 演示高一致性 PDF 建议用 Windows 本地环境 |
| 备份依赖本机 `mysqldump` | 未配置时不能创建真实备份 | 前端明确提示不可用，不生成伪记录 |

## 5. 知识库为空时的说明

当前系统已经具备知识库文档上传、条款解析和报告引用依据结构，但如果本地法规知识库为空，系统不应把 AI 的自由回答包装成严格法规判断。

演示或汇报时建议明确说明：

- 空知识库状态下，系统可以做图片事实识别、隐患描述和整改建议草稿。
- 空知识库状态下，正式法规依据、重大隐患定性和处罚依据都必须标记为“需人工复核”。
- 项目下一阶段核心工作是建立法规知识库数据底座，再把条文转化为规则，最后约束 AI 判断。

## 6. 下一阶段升级方向

后续路线按“法规知识库底座 -> 条文导入 -> 覆盖率看板 -> 规则库 -> AI 场景门禁 -> 规则驱动判断 -> 专业报告 v2 -> 人工复核闭环”推进。

优先级如下：

1. 固定 14 类安全生产法规分类。
2. 为法规文档和条款补充官方来源、发布机关、施行日期、现行状态和人工校验状态。
3. 提供 CSV/Excel 条文导入能力和小规模公开法规种子数据。
4. 增加知识库覆盖率看板，提示哪些分类可用于正式判断。
5. 建立人工维护的隐患规则库，重大隐患必须关联本地条文或判定标准。
6. AI 先判断图片是否属于安全生产检查场景，无关图片不生成正式报告。
7. AI 只抽取可见事实，隐患等级由规则和证据决定。
8. 报告升级为展示图片事实、命中规则、法规条款、证据充分性、整改建议和人工审核意见。

## 7. 阶段结论

当前版本已经适合做“系统功能完整度”和“工程治理能力”的阶段演示。它能够展示从登录、图片分析、报告生成到管理端治理的完整闭环。

但如果要把项目提升为具有商业说服力的安全生产专业系统，核心竞争力必须继续落在法规知识库、规则判定、依据追溯和人工复核闭环上。后续 PR13 到 PR23 正是围绕这条主线推进。
