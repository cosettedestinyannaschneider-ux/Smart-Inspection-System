# 发布与回归检查清单

本文档用于每次阶段合并、演示前、交付前做统一检查。它不保存任何真实数据库密码、AI API Key、证书或个人隐私数据。

## 1. 发布前基础检查

| 检查项 | 命令或操作 | 通过标准 | 状态 |
|---|---|---|---|
| 后端语法检查 | `npm --prefix project/backend run check` | 输出 `syntax-check passed` | 待检查 |
| 后端 mock 测试 | `npm --prefix project/backend test` | 全部测试通过，不调用真实 AI | 待检查 |
| 前端 H5 构建 | `npm --prefix project/uni-app-frontend run build:h5` | 构建完成且无阻断错误 | 待检查 |
| Docker 后端镜像 | `docker build -t safety-inspection-backend ./project/backend` | 镜像构建成功 | 待检查 |
| Docker 前端镜像 | `docker build -t safety-inspection-frontend ./project/uni-app-frontend` | 镜像构建成功 | 待检查 |
| Git 状态 | `git status --short` | 仅包含本次计划内变更 | 待检查 |

## 2. 配置与密钥检查

| 检查项 | 通过标准 | 状态 |
|---|---|---|
| `.env` 文件 | 未进入 Git 暂存区或提交记录 | 待检查 |
| 数据库密码 | 不出现在代码、文档、CI 明文配置中 | 待检查 |
| AI API Key | 只存在于本地 `.env` 或数据库加密字段中，不出现在仓库 | 待检查 |
| `JWT_ACCESS_SECRET` | 本地配置存在，仓库只保留示例占位 | 待检查 |
| `MODEL_CONFIG_SECRET` | 本地配置存在，仓库只保留示例占位 | 待检查 |
| `MYSQLDUMP_BIN` | 仅备份功能需要，未配置时页面应提示不可用 | 待检查 |

## 3. 数据库变更检查

| 检查项 | 通过标准 | 状态 |
|---|---|---|
| 是否涉及 DDL | PR 描述中明确写明 | 待检查 |
| `schema.sql` | DDL 变更已同步完整表结构 | 待检查 |
| `schemaInit.js` | 旧库兼容迁移已同步，或明确需要手动 SQL | 待检查 |
| 演示数据 | 只使用公开、脱敏、可重建数据 | 待检查 |
| 本地数据库 | 不因普通 PR 合并而删除重建 | 待检查 |

## 4. 核心业务回归

| 流程 | 验证步骤 | 通过标准 | 状态 |
|---|---|---|---|
| 登录回归 | 管理员和普通用户分别登录 | 登录成功后不闪退，角色分流正确 | 待检查 |
| 权限回归 | 普通用户访问管理员接口 | 后端拒绝，前端不展示越权入口 | 待检查 |
| 企业信息 | 普通用户查看和维护企业资料 | 只操作当前用户归属数据 | 待检查 |
| 图片上传 | 上传 1 张和多张隐患图片 | 图片预览、列表、删除正常 | 待检查 |
| AI 分析 | 使用已启用模型分析图片 | 有等待反馈，失败时有错误提示 | 待检查 |
| 报告生成 | 完成分析后下载 Word/PDF | 两种文件均可通过受控接口访问 | 待检查 |
| 历史记录 | 查看报告历史和会话记录 | 报告、图片、引用依据展示正常 | 待检查 |
| 管理员模型配置 | 新增、检测、启用、删除非启用模型 | API Key 脱敏，不泄露明文 | 待检查 |
| 日志工作台 | 查看工作台统计和操作日志 | 数据来自真实后端，不显示假趋势 | 待检查 |
| 模板管理 | 上传、设默认、下载模板 | 默认模板保护生效 | 待检查 |
| 知识库管理 | 上传、归档、查看条款 | 空知识库和解析失败都有明确提示 | 待检查 |
| 手动备份 | 配置 `MYSQLDUMP_BIN` 后创建备份 | 未配置时提示不可用，不造假记录 | 待检查 |

## 5. AI 与知识库风险回归

| 检查项 | 通过标准 | 状态 |
|---|---|---|
| 非业务图片 | 不应硬套正式隐患报告，应提示无法形成安全检查结论 | 待检查 |
| 无关文本问题 | 应拒绝业务范围外闲聊或引导回安全检查场景 | 待检查 |
| 知识库为空 | 不应声称已依据本地法规条文作出确定结论 | 待检查 |
| 法规依据 | 报告中展示的依据应能回查本地条文快照 | 待检查 |
| 重大隐患 | 后续规则库完成前，重大隐患结论应保守并提示人工复核 | 待检查 |

## 6. GitHub PR 合并前检查

| 检查项 | 通过标准 | 状态 |
|---|---|---|
| PR 描述 | 使用统一模板，写清变更、验证、风险 | 待检查 |
| CI 状态 | 后端检查、后端测试、前端构建、Docker 构建通过 | 待检查 |
| 分支基线 | 从最新 `teamrepo/main` 创建，不基于旧分支继续开发 | 待检查 |
| 合并策略 | 确认安全后合并，合并后可删除任务分支 | 待检查 |
| 本地同步 | 合并后按 `LOCAL_DEV_SYNC.md` 拉取主线，不重配 `.env` | 待检查 |
